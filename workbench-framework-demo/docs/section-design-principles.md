# Section 容器结构设计原则

## 核心理念

容器的一致性是规则的一致性，不是外观的一致性。同一套规则下产生的不同形态，本身就是一致的。

---

## 一、三级容器层级体系

页面内容区采用三级容器层级，最大嵌套不超过 3 层。

### L1 — Section（区域）

- **定位**：页面的一级内容分区，代表一个独立的业务域
- **标题位置**：在容器外部
- **标题样式**：Heading/3，不带 Icon，搭配折叠控件
- **内容容器**：填充 ColorBgBase（白色），描边 1px colorSplit，圆角 borderRadiusLG（主题定义为 16px）
- **滚动行为**：标题行支持 Sticky 吸顶，滚动进入该 Section 时自动悬停，滚出后恢复
- **Section 间距**：Section 之间的垂直间距（Gap）为 32px
- **内边距**：内容容器 Padding 为 32px

**识别条件（满足任意两条即为 L1）：**

1. 映射了一个独立的数据实体或 API 接口
2. 拥有独立的读写权限
3. 可以独立地呈现完成 / 未完成状态
4. 在不同业务场景下可能整体出现或消失

> 💬 和产品经理沟通时可以这样说：如果这块内容将来可能单独拆成一个独立的 Tab 页而不影响其他部分，那它就是一个 Section。

### L2 — Card（卡片）

- **定位**：Section 内部的可重复子实例
- **标题位置**：在容器内部
- **标题样式**：Heading/5 + 元信息（ID、状态等）+ 折叠控件
- **容器样式**：有边框，用于明确每个实例的边界范围

**识别条件（满足任意一条即需要 L2 卡片）：**

1. Section 内有多个同类子实例（如多个 SKU、多个规则组）
2. 子实例可以被独立折叠 / 展开
3. 子实例可以被增加 / 删除 / 拖拽重排

**不需要 L2 卡片的情况：**

当 Section 内只有一组固定内容、不会重复出现、没有子实例需要独立折叠或增删时，内容直接平铺在 Section 内容区，不包卡片。

> 💬 和产品经理沟通时可以这样说：如果用户会把这块内容理解为一张可以翻开 / 合上的卡片，并且标题行上有动态信息（状态、计数、ID），那就需要 L2 卡片。

### L3 — Field Group（字段分组）

- **定位**：Card 或 Section 内容区内的逻辑字段分组
- **标题位置**：无独立容器，仅为加粗标题文本
- **标题样式**：Base/Strong，与下方字段之间靠间距分隔
- **行为**：不可折叠、不可删除、不可拖拽

**识别标准：** 如果去掉这个标题，用户依然能通过字段本身理解上下文，那它就是 L3。

## 二、Section 是否包卡片？

### 不包卡片（简单 Section）

满足以下全部条件：
- Section 内只有一组固定内容，不会重复出现
- 没有子实例需要独立折叠 / 删除 / 拖拽排序

**骨架结构：**

```
┌─────────────────────────────────────────────┐
│  L1 Section 标题（Heading/3）        ▲ 折叠  │  ← 标题在容器外，不带 Icon
└─────────────────────────────────────────────┘
┌─────────────────────────────────────────────┐  ← ColorBgBase 填充，1px colorSplit 描边，圆角 16px
│                 Padding 32px                │
│  L3 字段分组标题（Base/Strong）               │
│  ┌─────────────────┐ ┌─────────────────┐   │
│  │  表单字段         │ │  表单字段         │   │
│  └─────────────────┘ └─────────────────┘   │
│                                             │
│  L3 字段分组标题（Base/Strong）               │
│  ┌─────────────────┐ ┌─────────────────┐   │
│  │  表单字段         │ │  表单字段         │   │
│  └─────────────────┘ └─────────────────┘   │
│                                             │
└─────────────────────────────────────────────┘

            ↕ Section 间距 Gap 32px

┌─────────────────────────────────────────────┐
│  L1 Section 标题（Heading/3）        ▲ 折叠  │
└─────────────────────────────────────────────┘
┌─────────────────────────────────────────────┐
│                 Padding 32px                │
│  ...                                        │
└─────────────────────────────────────────────┘

层级：L1 → L3
最大嵌套：2 层
```

### 包卡片（实例 Section）

满足以下任意一条：
- Section 内有多个同类子实例
- 子实例可独立折叠 / 展开
- 子实例可增加 / 删除 / 拖拽

**骨架结构：**

```
┌─────────────────────────────────────────────┐
│  L1 Section 标题（Heading/3）        ▲ 折叠  │  ← 标题在容器外，不带 Icon
└─────────────────────────────────────────────┘
                                                  ← ColorBgBase 填充，1px colorSplit 描边，圆角 16px
  ┌───────────────────────────────────────────┐
  │ ▲ L2 实例标题（Heading/5） ID: xxx · 状态  │  ← L2 卡片，标题在容器内
  │                                           │
  │  L3 字段分组标题（Base/Strong）              │
  │  ┌────────────────┐ ┌────────────────┐   │
  │  │  表单字段        │ │  表单字段        │   │
  │  └────────────────┘ └────────────────┘   │
  │                                           │
  │  L3 字段分组标题（加粗）                     │
  │  ┌─────────────────────────────────────┐ │
  │  │  表单字段                             │ │
  │  └─────────────────────────────────────┘ │
  │                                           │
  │  ┌─────────────────────────────────────┐ │  ← 功能性容器（圈定可拖拽区域）
  │  │  从属子列表标题（13px 加粗）           │ │
  │  │  ┌──────────────────────────────┐  │ │
  │  │  │ ① 列表项（名称/ID/标签）  ≡   │  │ │
  │  │  ├──────────────────────────────┤  │ │
  │  │  │ ② 列表项（名称/ID/标签）  ≡   │  │ │
  │  │  ├──────────────────────────────┤  │ │
  │  │  │ ③ 列表项（名称/ID/标签）  ≡   │  │ │
  │  │  └──────────────────────────────┘  │ │
  │  └─────────────────────────────────────┘ │
  │                                           │
  └───────────────────────────────────────────┘

  ┌───────────────────────────────────────────┐
  │ ▼ L2 实例标题（折叠态）   ID: xxx · 状态   │
  └───────────────────────────────────────────┘

  ┌───────────────────────────────────────────┐
  │ ▼ L2 实例标题（折叠态）   ID: xxx · 状态   │
  └───────────────────────────────────────────┘

层级：L1 → L2 → L3
最大嵌套：3 层
```

## 三、L3 内的从属关系表达

当 L3 字段分组下存在从属子列表时，根据子列表是否有整体性交互行为，选择不同的表达方式：

### 方式 A — 功能性容器（推荐用于有交互行为的子列表）

当子列表存在拖拽排序、批量选择、批量删除等整体性交互时，使用轻量边框容器包裹。此容器的目的不是表达层级，而是圈定交互范围。

- 容器使用 1px border-tertiary 边框，圆角与 L2 卡片一致
- 容器内标题降级为 13px 加粗，视觉权重低于 L3 标题
- 容器与上方字段之间保持 12–16px 间距

```
  L3 字段分组标题（14px 加粗）
  ┌─────────────────────────────────────┐
  │  表单字段                             │
  └─────────────────────────────────────┘

  ┌─────────────────────────────────────┐  ← 功能性容器（圈定可拖拽区域）
  │  从属子列表标题（13px 加粗）           │
  │  ┌─────────────────────────────┐    │
  │  │ ① 列表项（名称/ID/标签）  ≡  │    │
  │  ├─────────────────────────────┤    │
  │  │ ② 列表项（名称/ID/标签）  ≡  │    │
  │  ├─────────────────────────────┤    │
  │  │ ③ 列表项（名称/ID/标签）  ≡  │    │
  │  └─────────────────────────────┘    │
  └─────────────────────────────────────┘
```

**使用条件（满足任一）：**
- 子列表支持拖拽排序，需要明确的可操作区域边界
- 子列表内容信息密度高，缩进 + 竖线不足以提供清晰的视觉收束
- 子列表可整体增删（如"添加一组" / "清空全部"）

### 方式 B — 缩进 + 左侧竖线（适用于纯展示的从属内容）

当子列表为纯展示、无整体性交互行为时，使用缩进 + 竖线表达从属，不增加容器。

- 子列表整体左缩进 20px
- 左侧添加 2px 竖线（使用 border-tertiary 颜色）
- 子列表标题降级为 13px，视觉权重低于 L3 标题

```
  L3 字段分组标题（14px 加粗）
  ┌───────────────────────┐
  │  表单字段               │
  └───────────────────────┘
     │
     │  从属子列表标题（13px）       ← 缩进 + 竖线
     │  内容项 A
     │  内容项 B
     │  内容项 C
```

## 四、分隔线使用原则

### 使用分隔线的场景

- 无标题的同类重复项之间（如拖拽排序列表中的每个条目）
- 单个条目包含多行、多类型信息（标题 + 元数据 + 标签组），需要明确边界

### 不使用分隔线的场景

- 相邻两个内容块各自有明确的 L3 标题时，间距即分隔，不加横线

### 分隔线的层级区分

不同层级的分隔线应有不同的视觉权重。列表项之间的分隔线应比分组分隔线更轻量（如缩短宽度、降低颜色深度），避免所有横线看起来一样。

---

## 五、标题规则

所有层级的标题均不带 Icon，通过字号层级区分：

| 层级 | Token | 说明 |
|------|-------|------|
| L1 Section 标题 | Heading/3 | 页面一级分区标题，搭配折叠控件 |
| L2 Card 标题 | Heading/5 | 实例卡片标题，搭配 ID、状态等元信息 |
| L3 Field Group 标题 | Base/Strong | 字段分组标题，仅加粗文本 |

---

## 六、L1 标题 Sticky 吸顶规则

- 当用户向下滚动进入某个 Section 的内容区时，该 Section 的 L1 标题行自动吸顶
- 当滚动超出该 Section 的内容范围后，标题恢复原位
- 解决长内容 Section（如多个 SKU 展开时）用户失去上下文的问题
- 不增加任何额外的容器嵌套层级

---

## 七、应用示例

以 Combo 产品策略配置页为例：

```
Display strategy（L1，固定内容，不包卡片）
├── Calendar merge（L3）
└── Display placement（L3）

Operation & Fencing strategy（L1，固定内容，不包卡片）
├── Responsible BD（L3）
├── Auto publish/unpublish（L3）
└── Fencing setting（L3）

Selling & Fulfilment strategy（L1，多实例，包卡片）
├── Combo SKU #1 Adult ticket（L2 卡片）
│   ├── Selling strategy（L3）
│   └── Fulfilment strategy（L3）
│       └── Fulfilment sequence（L3 从属子列表）
├── Combo SKU #2 Child ticket（L2 卡片，折叠态）
└── Combo SKU #3 Senior ticket（L2 卡片，折叠态）
```
