# 导航内容输出格式说明（对齐 B 端架构与导航设计规范）

本文档将《B端架构与导航设计全景规范手册》中的 **L1/L2 侧栏规则** 与当前项目的导航配置格式对齐，供各业务按规范梳理导航内容并交付配置。

---

## 一、层级与项目中的对应关系

| 规范层级 | 含义 | 在本项目中的位置 | 本文档范围 |
|----------|------|------------------|------------|
| **L0 场域** | 逻辑隔离的子系统（如运营中心） | 左上角展开后的弹窗切换器 | ❌ 暂不约定 |
| **L1 对象** | 物理实体分类，侧栏一级入口 | **左侧边栏一级菜单**（点击后可选展开二级） | ✅ 以下从 L1 开始 |
| **L2 任务** | 具体业务操作线，侧栏二级入口 | **左侧边栏二级菜单**（在一级下的面板中） | ✅ |
| L3 状态 | 任务内进度切片 | 页面内顶部 Tab | 不在导航配置中 |
| L4 筛选 | 业务数据细分 | 页面内筛选区 | 不在导航配置中 |

**交付物**：只约定 **L1（侧栏一级）+ L2（侧栏二级）** 的配置格式；L0 弹窗内容、L3 Tab、L4 筛选由各业务页面自行实现。

---

## 二、规范中的 L1 / L2 规则（直接引用手册）

### L1 侧栏一级（对象 / Object）

- **逻辑定义**：物理实体分类，解决“正在处理的业务主体是谁”。
- **命名规范**：**纯名词**（如：商户管理、合同管理）。
- **数量上限**：**≤ 10 个**（同一 L0 场域下）。
- **认知目标**：锁定正在处理的业务主体。

### L2 侧栏二级（任务 / Mission）

- **逻辑定义**：具体业务操作线，解决“当前执行的任务性质是什么”。
- **命名规范**：**动宾结构**（如：资质审核、合同审批）。
- **数量上限**：**每个 L1 下 ≤ 8 个**。
- **认知目标**：明确当前执行的任务性质。

### 判定原则（L1 vs L2）

- **独立性原则**：该实体是否具有独立生命周期？→ 是则为 **L1 对象**（如：商户、合同）。
- **归属感原则**：该功能离开主对象是否还有意义？→ 否则为 **L2 任务**（如：商户下的资质审核）。
- **跨模块原则**：该功能是否需聚合多个对象的数据（如：全平台审批中心）？→ 是则提升为 **L1 对象**。

### 交互禁区（手册红线）

- **严禁状态上移**：侧栏中禁止出现进度/状态词（如“待审核”“进行中”），避免导航与内容区“折返跑”。
- **严禁名词污染**：导航文案禁止使用“我的”“超级”“一键”等装饰性前缀。
- **严禁场域混杂**：L0 切换后，下层所有状态需物理重置（本配置仅描述 L1/L2 结构，重置逻辑由框架/业务实现）。

## 三、与当前项目导航的映射

| 规范 | 本项目配置结构 | 说明 |
|------|----------------|------|
| L1 一级入口（≤10） | `groups[].items[]` 中**每个 item** | 一个 item = 一个侧栏一级菜单项，可带 `children` 表示有二级 |
| L2 二级入口（每 L1 ≤8） | `items[].children[]` | 每个 child = 该 L1 下的一个侧栏二级菜单项 |
| 分组展示（可选） | `groups[].groupLabel` | 如「MENU」「GENERAL」，用于侧栏分组标题；**不改变 L1 数量上限**（所有 groups 的 items 合计 ≤10） |

**数量约束汇总**：

- 所有 `groups` 下 `items` 的总数：**≤ 10**（即 L1 总数 ≤ 10）。
- 每个 `items[].children` 的长度：**≤ 8**（即每个 L1 下 L2 ≤ 8）。

---

## 四、输出格式（JSON）

### 4.1 结构说明

- **groups**：数组，每个元素为一组侧栏入口，有分组标题（groupLabel）和一组 L1 项（items）。
- **groupLabel**： string，侧栏分组标题（如 "MENU"、"GENERAL"），仅展示用。
- **items**：数组，**L1 列表**；所有 groups 的 items 合计 ≤10。
  - **key**：string，唯一标识，用于选中态与路由/内容映射；建议小写+连字符（如 `account-management`）。
  - **label**：string，**L1 展示文案，须符合「纯名词」**。
  - **icon**：string，可选。图标文件名（不含 `.svg`）或以 `/` 开头的完整路径，见 4.3。
  - **children**：数组，可选，**L2 列表**；长度 ≤8。
    - **key**：string，唯一标识。
    - **label**：string，**L2 展示文案，须符合「动宾结构」**。

### 4.2 JSON Schema 示例

```json
{
  "groups": [
    {
      "groupLabel": "MENU",
      "items": [
        {
          "key": "my-bench",
          "label": "工作台",
          "icon": "dashboard"
        },
        {
          "key": "merchant-management",
          "label": "商户管理",
          "icon": "merchant-management",
          "children": [
            { "key": "qualification-review", "label": "资质审核" },
            { "key": "contract-approval", "label": "合同审批" }
          ]
        }
      ]
    }
  ]
}
```

### 4.3 图标约定

- **icon 为「文件名」**（不含 `.svg`、不以 `/` 开头）：  
  对应静态资源路径：`/klook-bench/nav-icons/{icon}.svg`。  
  请将 SVG 放入项目 `public/klook-bench/nav-icons/`，文件名与 `icon` 一致。
- **icon 为「以 `/` 开头的路径」**：  
  直接作为图标 URL 使用（如 `/custom/icons/foo.svg`）。

### 4.4 key 命名建议

- **L1 / L2 的 key**：建议全应用唯一、小写、连字符（如 `account-acquisition`、`lead-management`），便于路由与内容映射。
- **label**：仅用于展示，需满足规范（L1 纯名词，L2 动宾），不参与路由解析。

---

## 五、交付清单与校验

业务方按上述规则梳理 L1/L2 后，交付：

1. **一份 JSON 文件**：符合 4.1–4.2 的结构；  
   - 所有 groups 的 items 总数 ≤10；  
   - 每个 item 的 children 数量 ≤8；  
   - L1 label 为纯名词，L2 label 为动宾结构；  
   - 无状态词、无“我的/超级/一键”等前缀。
2. **一组 SVG 图标**（若使用文件名方式）：放入 `public/klook-bench/nav-icons/`，文件名与配置中 `icon` 一致。

**替换方式**：将本项目的默认配置 `src/config/nav.config.json` 替换为上述 JSON（或通过构建/环境变量指向该文件），并更新 `public/klook-bench/nav-icons/` 中的图标，即可完成 L1+L2 侧栏的整批替换，无需改框架代码。

## 六、与规范手册的对照表

| 手册条款 | 本格式中的体现 |
|----------|----------------|
| L1 侧栏一级、纯名词、≤10 | `groups[].items[]`，items 合计 ≤10，label 按纯名词梳理 |
| L2 侧栏二级、动宾、每 L1 ≤8 | `items[].children[]`，每个 children 长度 ≤8，label 按动宾梳理 |
| 严禁状态上移 | 校验：label 中不出现“待审核”“进行中”等状态词 |
| 严禁名词污染 | 校验：label 中不使用“我的”“超级”“一键”等前缀 |
| L0 场域 | 不在此配置中，对应左上角弹窗，暂不约定 |

以上为从 **L1 侧栏** 开始的导航内容输出格式说明，与《B端架构与导航设计全景规范手册》中的 L1/L2 规则一致，可直接用于各业务导航内容梳理与交付。
