# 03b · 页面内容容器结构（L1 / L2 / L3）

容器的一致性是规则的一致性，不是外观的一致性。同一套规则下产生的不同形态，本身就是一致的。

---

## 前置依赖

本规则的判断有两条路径：

| 路径 | 前提条件 | 执行方式 |
|------|---------|---------|
| **A — 有结构化 PRD** | 产品经理提供了清晰的结构化 PRD，能说明页面的层级关系 | 直接读取 PRD 中的层级信息，映射为 L1/L2/L3，输出结构建议 |
| **B — 无结构化 PRD** | 只有功能描述或口头需求 | 使用下方识别条件自行判断层级，输出结构建议 + 标记不确定项 |

路径 A 优先于路径 B。当 PRD 层级信息与识别条件冲突时，以 PRD 为准，但需标注冲突点供评审确认。

---

## 一、三级容器层级体系

页面内容区采用三级容器层级，最大嵌套不超过 3 层。

### L1 — Section（区域）

- **定位**：页面的一级内容分区，代表一个独立的业务域
- **标题位置**：在容器外部
- **标题样式**：Heading/3，不带 Icon，搭配折叠控件
- **内容容器**：填充 ColorBgBase（白色），描边 1px colorSplit，圆角 borderRadiusLG（16px）
- **滚动行为**：标题行支持 Sticky 吸顶
- **Section 间距**：32px
- **内边距**：32px

**识别条件（满足任意两条即为 L1）：**

1. 映射了一个独立的数据实体或 API 接口
2. 拥有独立的读写权限
3. 可以独立地呈现完成 / 未完成状态
4. 在不同业务场景下可能整体出现或消失

> 和产品经理沟通时：如果这块内容将来可能单独拆成一个独立的 Tab 页而不影响其他部分，那它就是一个 Section。

### L2 — Card（卡片）

- **定位**：Section 内部的可重复子实例
- **标题位置**：在容器内部
- **标题样式**：Heading/5 + 元信息（ID、状态等）+ 折叠控件
- **容器样式**：有边框，用于明确每个实例的边界范围

**识别条件（满足任意一条即需要 L2 卡片）：**

1. Section 内有多个同类子实例（如多个 SKU、多个规则组）
2. 子实例可以被独立折叠 / 展开
3. 子实例可以被增加 / 删除 / 拖拽重排

**不需要 L2 卡片的情况：**

当 Section 内只有一组固定内容、不会重复出现、没有子实例需要独立折叠或增删时，内容直接平铺在 Section 内容区，不包卡片。

> 和产品经理沟通时：如果用户会把这块内容理解为一张可以翻开 / 合上的卡片，并且标题行上有动态信息（状态、计数、ID），那就需要 L2 卡片。

### L3 — Field Group（字段分组）

- **定位**：Card 或 Section 内容区内的逻辑字段分组
- **标题位置**：无独立容器，仅为加粗标题文本
- **标题样式**：Base/Strong，与下方字段之间靠间距分隔
- **行为**：不可折叠、不可删除、不可拖拽

**识别标准**：如果去掉这个标题，用户依然能通过字段本身理解上下文，那它就是 L3。

## 二、Section 是否包卡片？

### 不包卡片（简单 Section）

满足以下全部条件：
- Section 内只有一组固定内容，不会重复出现
- 没有子实例需要独立折叠 / 删除 / 拖拽排序

```
L1 Section 标题（Heading/3）                    ▲ 折叠
┌─────────────────────────────────────────────────┐
│  Padding 32px                                   │
│                                                 │
│  L3 字段分组标题（Base/Strong）                    │
│  ┌──────────────┐  ┌──────────────┐            │
│  │  表单字段      │  │  表单字段      │            │
│  └──────────────┘  └──────────────┘            │
│                                                 │
│  L3 字段分组标题（Base/Strong）                    │
│  ┌──────────────┐  ┌──────────────┐            │
│  │  表单字段      │  │  表单字段      │            │
│  └──────────────┘  └──────────────┘            │
└─────────────────────────────────────────────────┘

层级：L1 → L3（最大嵌套 2 层）
```

### 包卡片（实例 Section）

满足以下任意一条：
- Section 内有多个同类子实例
- 子实例可独立折叠 / 展开
- 子实例可增加 / 删除 / 拖拽

```
L1 Section 标题（Heading/3）                    ▲ 折叠
┌─────────────────────────────────────────────────┐
│  ┌─────────────────────────────────────────┐   │
│  │ ▲ L2 实例标题（Heading/5）  ID · 状态    │   │
│  │                                         │   │
│  │  L3 字段分组（Base/Strong）               │   │
│  │  ┌────────────┐  ┌────────────┐        │   │
│  │  │  表单字段    │  │  表单字段    │        │   │
│  │  └────────────┘  └────────────┘        │   │
│  └─────────────────────────────────────────┘   │
│  ┌─────────────────────────────────────────┐   │
│  │ ▼ L2 实例标题（折叠态）  ID · 状态        │   │
│  └─────────────────────────────────────────┘   │
└─────────────────────────────────────────────────┘

层级：L1 → L2 → L3（最大嵌套 3 层）
```

## 三、L3 内的从属关系表达

当 L3 字段分组下存在从属子列表时：

### 方式 A — 功能性容器（有交互行为的子列表）

**使用条件（满足任一）：**
- 子列表支持拖拽排序
- 子列表信息密度高，缩进不足以提供清晰收束
- 子列表可整体增删

容器：1px border-tertiary 边框，圆角与 L2 一致。容器内标题 13px 加粗。

### 方式 B — 缩进 + 左侧竖线（纯展示的从属内容）

无整体性交互行为时使用。左缩进 20px，左侧 2px 竖线 border-tertiary 颜色。

---

## 四、分隔线使用原则

- **使用**：无标题的同类重复项之间；单个条目包含多行多类型信息时
- **不使用**：相邻内容块各有 L3 标题时，间距即分隔
- **层级区分**：列表项分隔线比分组分隔线更轻量（缩短宽度或降低颜色深度）

---

## 五、标题规则

| 层级 | Token | 说明 |
|------|-------|------|
| L1 Section | Heading/3 | 页面一级分区，搭配折叠控件 |
| L2 Card | Heading/5 | 实例卡片，搭配 ID、状态等元信息 |
| L3 Field Group | Base/Strong | 字段分组，仅加粗文本 |

所有层级标题不带 Icon，通过字号层级区分。

---

## 六、L1 标题 Sticky 吸顶规则

- 用户向下滚动进入 Section 内容区时，L1 标题行自动吸顶
- 滚动超出 Section 范围后恢复原位
- 解决长内容 Section 用户失去上下文的问题
- 不增加额外容器嵌套层级

---

## 七、应用示例

以 Combo 产品策略配置页为例：

```
Display strategy（L1，固定内容，不包卡片）
├── Calendar merge（L3）
└── Display placement（L3）

Operation & Fencing strategy（L1，固定内容，不包卡片）
├── Responsible BD（L3）
├── Auto publish/unpublish（L3）
└── Fencing setting（L3）

Selling & Fulfilment strategy（L1，多实例，包卡片）
├── Combo SKU #1 Adult ticket（L2 卡片）
│   ├── Selling strategy（L3）
│   └── Fulfilment strategy（L3）
│       └── Fulfilment sequence（L3 从属子列表）
├── Combo SKU #2 Child ticket（L2 卡片，折叠态）
└── Combo SKU #3 Senior ticket（L2 卡片，折叠态）
```
