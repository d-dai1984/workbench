# 07 · 状态 · 反馈 · 权限 · 异常

高风险操作必须定义确认、反馈和回退策略，不允许在缺少状态定义时直接给出页面方案。

---

## 状态定义规则

每个列表页的数据状态必须在输出前明确定义：

| 要素 | 说明 |
|------|------|
| 状态名称 | 用业务语言命名，如「待审核」「已上线」「已关闭」 |
| 状态颜色 | 使用 antd Tag preset（`green` / `red` / `blue` / `default`） |
| 可执行操作 | 每种状态下允许的操作列表 |
| 流转条件 | 从哪种状态可以转到哪种状态 |

**规则**：
- 状态必须颜色 + 文字双重标识，不单独用颜色区分
- 状态数量 > 5 种需提供流转图
- 不允许在未定义状态集合时给出列表页方案

---

## 操作反馈规则

| 场景 | 组件 | 规格 |
|------|------|------|
| 轻量操作结果（保存、删除成功） | `message` | 顶部居中，`duration: 2s`，自动消失 |
| 需持续关注（异步任务、通知） | `notification` | 右上角，`duration: 4.5s` 或手动关闭 |
| 页面内持续状态（警告、提示） | `Alert` | inline，不自动消失 |
| 表单字段错误 | Form inline 错误 | 字段下方，不用 `message` |

**规则**：
- 静态方法（`message.xxx` / `Modal.xxx`）必须在根节点使用 `<App>` 包裹
- 操作成功后必须有反馈，不允许无声成功
- 操作失败必须说明原因，不允许只显示「操作失败」

---

## 确认策略

| 风险等级 | 场景 | 组件 | 规格 |
|---------|------|------|------|
| 低 | 单条删除、普通操作 | `Popconfirm` | 内联确认气泡 |
| 中 | 批量操作、状态批量变更 | `Modal.confirm` | 说明影响范围 |
| 高 | 不可逆操作、数据清除 | `Modal.confirm` | `okButtonProps: { danger: true }` + 明确后果说明 |

**规则**：
- 确认按钮用明确动词（「删除」「提交」），不用「确定」
- 高风险 Modal 必须说明「该操作不可撤销」
- 批量操作确认需展示「已选 N 条」数量

---

## 加载状态规则

| 场景 | 方案 |
|------|------|
| 首屏页面加载 | `Skeleton` 占位 |
| 操作等待（提交、查询） | 局部 `Spin`，`delay={300}` |
| 表格数据加载 | `Table` 的 `loading` prop |
| 按钮操作中 | `Button loading` prop，禁止重复点击 |

**规则**：
- `Spin delay={300}`：等待超过 300ms 才显示，避免闪烁
- 加载期间禁用可能引发重复提交的操作按钮

---

## 空状态规则

插画资源来自 Klook 设计库（Ant x Klook library），使用 `Status/` 系列插画，文档：https://www.figma.com/design/n72R5lEBMi3wVlKWzJ77EA

| 场景 | 插画组件 | 说明文字 | 操作入口 |
|------|---------|---------|---------|
| 列表无数据 | `Status/Nodata` | 自定义说明 | 「去新建」或相关操作 |
| 搜索无结果 | `Status/SearchNodata` | 「No results found」 | 清除筛选条件 |
| 无权限（403） | `Status/Inaccessible403` | 权限说明 | 「Contact admin」 |
| 网络断开 | `Status/Disconect` | 「Connection lost」 | 「Retry」按钮 |
| 服务器错误（500） | `Status/ServerError500` | 「Something went wrong」 | 「Retry」按钮 |
| 404 | `Status/404` | 「Page not found」 | 返回上一页 |

**规则**：
- 不使用 antd 默认 `Empty` 组件，统一使用 Klook 设计库 `Status/` 系列插画
- 空状态必须提供下一步操作入口，不允许死路
- 说明文字使用英文（产品为英文界面）

## 权限规则

| 权限粒度 | 处理方式 |
|---------|---------|
| 页面级无权限 | 展示权限空状态页（插图 + 说明 + 联系管理员） |
| 操作级无权限 | 按钮置灰 + `Tooltip` 说明原因 |
| 数据级部分无权限 | 字段显示「—」或「无权限」标注 |

**规则**：
- 权限判断在渲染层处理，不在业务逻辑层
- 无权限状态必须有明确说明，不能只显示空白或 404
- 不把权限逻辑硬编码进视觉组件

---

## 异常处理规则

| 异常类型 | 处理方式 |
|---------|---------|
| 网络超时 / 请求失败 | `notification.error` + 重试入口 |
| 表单提交失败（业务错误） | Drawer / Modal 不关闭，inline 展示错误 |
| 页面级数据加载失败 | `Result` 组件（`status="error"`）+ 返回上一页 |
| 404 / 500 | `Result` 组件对应 status |

**规则**：
- 任何操作失败都不能无声失败
- 表单提交失败不关闭弹窗，保留用户输入
- 页面级错误必须提供恢复路径（重试 / 返回 / 联系支持）
