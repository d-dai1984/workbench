# 04 · 业务模块规则

业务模块是可复用的页面组成单元，每个模块有固定结构和约束，不允许随意变形。

---

## 筛选表格模块

**适用**：列表页的核心模块

**筛选区两种形态**：

| 形态 | 适用场景 | 结构 |
|------|---------|------|
| 平铺式 | 筛选维度固定、字段较少（≤ 4 个） | Form 表单，字段直接展开在表格上方 |
| 收起式 | 筛选维度多、以关键词搜索为主入口 | 搜索框 + Advanced Filters 按钮，点击展开更多条件 |

**结构（平铺式）**：
```
筛选区（Form，独立一行）
  └── 筛选字段（≤ 4 个常显，超出放「更多筛选」）
  └── 操作按钮（搜索 + 重置，右对齐）
操作区（新建/导出，右对齐）
Table
  └── 数据列（状态列颜色 + 文字双重标识）
  └── 操作列（固定最右，全部平铺）
Pagination（右对齐，显示总数）
```

**结构（收起式）**：
```
筛选区
  └── 搜索框（支持多字段关键词）
  └── Advanced Filters 按钮（展开更多条件）
数据统计行（显示当前结果数量）
Table / Card List
  └── 操作列（固定最右，全部平铺）
Pagination（右对齐，显示总数）
```

规则：
- 平铺式常显字段不超过 4 个，超出放「更多筛选」收纳面板
- 表格 `rowKey` 必须是稳定唯一值（用 `id`，不用 `index`）
- 排序/筛选/分页统一走 `Table onChange`
- 数据量 > 5000 行时启用虚拟滚动（`virtual` prop）

## 批量操作模块

**适用**：列表页需要多选并批量处理数据时

**结构**：
```
勾选 → 批量操作栏浮现（表格上方）
  └── 已选 N 条 + 操作按钮组（批量删除/导出/状态变更等）
  └── 取消选择
```

规则：
- 批量操作栏只在有勾选项时显示
- 批量操作按钮不超过 4 个，超出折进 Dropdown
- 批量操作前必须有确认环节（Modal.confirm）
- 批量操作结果必须有反馈（成功/失败数量）

---

## 抽屉编辑模块

**适用**：在列表页或详情页内，不跳转页面地完成编辑

**结构**：
```
Drawer（placement="right"，inline=false）
  └── 标题（动词 + 对象名，如「编辑活动」）
  └── 表单内容（可滚动）
  └── 底部操作区（固定，取消 + 保存）
```

规则：
- 字段 ≤ 6 个用 Modal，字段 > 6 个用 Drawer
- Drawer 底部操作区固定，不随内容滚动（`footer` prop）
- 提交失败不关闭 Drawer，inline 展示错误
- 提交成功关闭 Drawer + 刷新列表

---

## 权限状态模块

**适用**：页面内存在权限控制的区域或操作

**结构**：
```
无权限访问整页 → 展示权限空状态页（插图 + 说明 + 联系管理员入口）
无权限操作某按钮 → 按钮置灰 + Tooltip 说明原因
部分数据无权限 → 数据行标注「无权限」或隐藏敏感字段
```

规则：
- 权限判断在页面级和操作级分别处理
- 无权限状态必须有明确说明，不能只显示空白
- 不把权限逻辑硬编码进视觉组件

---

## 状态标签模块

**适用**：列表页、详情页中对数据状态的可视化展示

规则：
- 状态标签优先使用 antd Tag preset（`green` / `red` / `blue` / `default`）
- 状态必须颜色 + 文字双重标识，不单独用颜色区分
- 同一页面的状态标签风格保持一致，不混用不同形态
- 状态数量 > 5 种时，需要提供状态说明文档
